# Docs Index

This folder contains strategy, product history, and top-level planning artifacts.

## Primary Sections

- `white-paper-alignment.md`: Concept-to-implementation mapping for ANAVI terminology.
- `ANAVI-*.md`: legacy and strategic PRD documents.
- `plans/`: dated strategy and implementation plans.
- `plans/PLAN_MANIFEST.md`: active/reference/superseded status map.
- `plans/NAMING_CONVENTIONS.md`: naming standard for all new plans.

## Usage Rules

- Add new strategic PRDs and cross-team plans here.
- Keep app-specific implementation logs in `anavi/docs/`.
- Do not delete old plans; mark them as superseded in plan indexes.
