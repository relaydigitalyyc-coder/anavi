# Dashboard Master Rollout PRD (C1/C2/C3)

**Date:** March 2, 2026  
**Owner:** Product + Engineering  
**Status:** Master rollout plan

## Objective

Roll out all dashboard upgrade tracks:
- C1 Dealmaker Command Center
- C2 Owner-Operator Execution
- C3 LP Reporting and Investor Experience

in a phased order that maximizes visible value quickly while controlling implementation risk.

## Rollout Order

1. **Phase A (Immediate Value): C1**
- Deliver command center and workflow throughput improvements first.
- Reason: biggest visible improvement to daily operator flow.

2. **Phase B (Trust + Execution): C2**
- Deliver principal execution reliability and disclosure control.
- Reason: directly impacts perceived platform safety for high-value raises.

3. **Phase C (Institutional Credibility): C3**
- Deliver LP/investor reporting and portal-grade polish.
- Reason: compounds trust and retention after core execution is solid.

## Cross-Phase Shared Requirements

- No decorative-only modules.
- Every module must map to a decision or action.
- Live proof must be fixture/data-backed, not static copy.
- Preserve whitepaper terminology and trust architecture concepts.

## Milestones

1. **M1**: Shared surface kit and interaction standards complete.
2. **M2**: Dealmaker and principal dashboards upgraded with live proof + CTA hierarchy.
3. **M3**: Investor/LP experience upgraded with institutional metrics and self-service framing.
4. **M4**: Demo narrative pass and issue hardening complete.

## Verification Matrix

- Engineering: `pnpm check`
- UX: desktop + mobile pass
- Narrative: 10-second “what now?” comprehension test per persona
- Risk: no disclosure-control regressions in principal flows

## Definition of Done

- All three PRD tracks delivered to planned scope.
- Persona-specific pages and dashboard cards show immediate value through software state.
- Demo can be run end-to-end for Originator, Investor, and Principal without gaps.
