# App Docs Index (`anavi/docs`)

This folder is the canonical app-local documentation surface.

## Structure

- `plans/`: feature and rollout plans used for implementation.
- `ops/`: active memory + task board for ongoing development.
- `todo_legacy_2026-02-15.md`: historical checklist (archived).

## Canonical Operational Files

- `ops/ENGINEERING_MEMORY.md`
- `ops/TODO_BOARD.md`
- `ops/OPERATOR_DASHBOARD.md`
- `ops/DOCUSIGN_INTEGRATION_BLUEPRINT.md`
- `ops/DOCUSIGN_OPERATIONS_RUNBOOK.md`
- `ops/DOCUSIGN_API_REFERENCE.md`

## Rules

- Update `ops/ENGINEERING_MEMORY.md` after major implementation batches.
- Keep `ops/TODO_BOARD.md` current with `Next Up`, `In Progress`, and `Done`.
- Keep plan-level status pointers updated in `plans/README.md`.
